# Security Policy

## Supported Versions

| Component                         | Version / Status               | Security Updates |
|-----------------------------------|--------------------------------|------------------|
| ACCX Wallet (CCXT integration)    | Latest `main` branch           | Active |
| ACCXBot (Telegram)                | Latest `main` branch           | Active |
| Walk-to-Earn contracts            | In development (not deployed)  | N/A |
| Staking contracts                 | Planned (Q1 2026)              | N/A |
| GameFi mini-apps                  | In development                 | N/A |

Only the current `main` branch of active repositories receives security updates.

## Reporting a Vulnerability

We take security seriously. If you discover any vulnerability (wallet, bot, contracts, website, etc.), please report it responsibly.

**DO NOT open public GitHub issues for security vulnerabilities.**

### How to Report
Send a detailed report to:  
**security@africacryptochainx.com**

Or contact us privately via:
- Telegram: @ACCX_Security_Lead (verified admin only)

### What to Include
- Description of the vulnerability
- 
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response Timeline
- Acknowledgment: within **24 hours**
- Initial triage: within **72 hours**
- Regular updates every 7 days until resolved

### Disclosure Policy
- We will credit you in release notes (unless you prefer anonymity)
- We request you allow us reasonable time to fix before public disclosure

Thank you for helping keep AfricaCryptoChainx safe.
